## Installation

Install the package and its peer dependencies:

<PackageInstaller command="installer -D @pelatform/eslint-config-vite eslint typescript" />

<Note>
This package requires ESLint 9.30.0 or greater and automatically includes `@pelatform/eslint-config-react` as a dependency.
</Note>

## Quick Start

<Steps>
  <Step title="Create ESLint Config">
    Create an `eslint.config.js` file in your project root:

    ```js eslint.config.js
    import { configs } from "@pelatform/eslint-config-vite";

    export default [
      ...configs.viteFast,
      {
        languageOptions: {
          parserOptions: {
            tsconfigRootDir: import.meta.dirname,
          },
        },
      },
    ];
    ```
  </Step>

  <Step title="Add Scripts to package.json">
    Add lint and lint:fix scripts to your package.json:

    ```json package.json
    {
      "scripts": {
        "lint": "eslint .",
        "lint:fix": "eslint . --fix"
      }
    }
    ```
  </Step>

  <Step title="Run Linting">
    Test your configuration:

    <PackageRun command="run lint" />
  </Step>
</Steps>

## Advanced Setup

<Accordion title="Custom Vite Rules">
  Customize Vite-specific rules:

  ```js eslint.config.js
  import { configs } from "@pelatform/eslint-config-vite";

  export default [
    ...configs.viteFast,
    {
      languageOptions: {
        parserOptions: {
          tsconfigRootDir: import.meta.dirname,
        },
      },
      rules: {
        "react-refresh/only-export-components": [
          "warn",
          { allowConstantExport: true, allowArrowFunction: false }
        ],
        "node/no-process-exit": "off",
      },
    },
  ];
  ```
</Accordion>

  <Accordion title="Framework-Specific Rules">
    Configure for specific Vite frameworks:

    ```js eslint.config.js
    import { configs } from "@pelatform/eslint-config-vite";
    import globals from "globals";

    export default [
      ...configs.viteFast,
      {
        languageOptions: {
        parserOptions: {
          tsconfigRootDir: import.meta.dirname,
        },
      },
    },
    {
      files: ["**/*.test.tsx", "**/*.spec.tsx"],
      rules: {
        "@typescript-eslint/no-explicit-any": "off",
        "react-hooks/rules-of-hooks": "off",
      },
    },
    {
      files: ["vite.config.ts", "vitest.config.ts"],
      languageOptions: {
        globals: {
          ...globals.node,
        },
      },
    },
  ];
  ```
</Accordion>

## Rules Overview

### Vite-Specific Rules

<Tabs>
  <Tab title="React Refresh">
    - `react-refresh/only-export-components`: Warn for non-component exports
    - Allows constant exports with HMR
    - Ensures proper component structure for fast refresh
  </Tab>
  <Tab title="TanStack Router">
    - `@tanstack/router/route-path-segment-casing`: Validate path segment casing
    - `@tanstack/router/route-path-param-case`: Validate parameter naming
    - `@tanstack/router/route-name-prefix`: Ensure consistent route naming
  </Tab>
  <Tab title="Environment Variables">
    - Validates proper usage of Vite environment variables
    - Ensures type-safe environment variable access
    - Prevents runtime errors from undefined variables
  </Tab>
</Tabs>

## Troubleshooting

<Accordion title="React Refresh Issues">
  Ensure proper component exports:

  ```tsx component.tsx
  // ✅ Default export
  export default function MyComponent() {
    return <div>Component</div>;
  }

  // ✅ Named export with component name
  export function NamedComponent() {
    return <div>Component</div>;
  }

  // ❌ Arrow function without name
  export const ArrowComponent = () => <div>Component</div>;
  ```
</Accordion>

<Accordion title="TanStack Router Rules">
  Router rules only apply to files importing the library:

  ```ts router.ts
  import { createRouter } from '@tanstack/react-router';
  // Router rules automatically apply
  ```
</Accordion>

<Accordion title="Environment Variables">
  Validate in Vite config:

  ```ts vite.config.ts
  export default defineConfig({
    define: {
      __APP_VERSION__: JSON.stringify(process.env.npm_package_version),
    },
  });
  ```
</Accordion>
