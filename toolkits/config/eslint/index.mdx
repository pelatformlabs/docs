## Installation

Install the package and its peer dependencies:

<PackageInstaller command="installer -D @pelatform/eslint-config eslint typescript" />

<Note>
This package requires ESLint 9.30.0 or greater since it uses the flat config system.
</Note>

## Quick Start

<Steps>
  <Step title="Create ESLint Config">
    Create an `eslint.config.js` file in your project root:

    ```js eslint.config.js
    import { configs } from "@pelatform/eslint-config";

    export default [
      ...configs.fast,
      {
        languageOptions: {
          parserOptions: {
            tsconfigRootDir: import.meta.dirname,
          },
        },
      },
    ];
    ```
  </Step>

  <Step title="Add Scripts to package.json">
    Add lint and lint:fix scripts to your package.json:

    ```json package.json
    {
      "scripts": {
        "lint": "eslint .",
        "lint:fix": "eslint . --fix"
      }
    }
    ```
  </Step>

  <Step title="Run Linting">
    Test your configuration:

    <PackageRun command="run lint" />
  </Step>
</Steps>

## Features

### TypeScript Integration

- **Strict Type Checking**: Full TypeScript support with type-aware linting
- **Modern Syntax**: Supports latest ECMAScript features and TypeScript syntax
- **Project Service**: Automatic TypeScript project detection and configuration
- **File-specific Rules**: Different rules for different file types (CJS, MJS, TSX)

### Linting Rules

- **Code Quality**: Comprehensive rules for maintainable and error-free code
- **Best Practices**: Industry-standard coding conventions
- **Security**: Security-focused rules to prevent common vulnerabilities
- **Performance**: Rules that promote efficient code patterns

### Monorepo Support

- **Turborepo Integration**: Pre-configured for Turborepo environments
- **Global Ignores**: Intelligent file ignoring for build artifacts and dependencies
- **Cross-repo Consistency**: Standardized linting across multiple packages

## Advanced Setup

<Accordion title="Custom Rules">
  Override specific rules:

  ```js eslint.config.js
  import { configs } from "@pelatform/eslint-config";

  export default [
    ...configs.fast,
    {
      languageOptions: {
        parserOptions: {
          tsconfigRootDir: import.meta.dirname,
        },
      },
      rules: {
        "@typescript-eslint/explicit-member-accessibility": "off",
        "prefer-const": "error",
      },
    },
  ];
  ```
</Accordion>

<Accordion title="File-Specific Rules">
  Configure rules for specific file patterns:

  ```js eslint.config.js
  import { configs } from "@pelatform/eslint-config";

  export default [
    ...configs.fast,
    {
      languageOptions: {
        parserOptions: {
          tsconfigRootDir: import.meta.dirname,
        },
      },
    },
    {
      files: ["**/*.test.ts", "**/*.spec.ts"],
      rules: {
        "@typescript-eslint/no-explicit-any": "off",
      },
    },
  ];
  ```
</Accordion>

<Accordion title="Biome Integration">
  Combine with Biome for optimal performance:

  1. Install Biome:

  <PackageInstaller command="installer -D @pelatform/biome-config @biomejs/biome" />

  2. Create Biome config:
  ```jsonc biome.jsonc
  {
    "$schema": "./node_modules/@biomejs/biome/configuration_schema.json",
    "extends": ["@pelatform/biome-config/base"]
  }
  ```

  3. Use the fast ESLint configuration
</Accordion>

## Examples

### Basic Setup

```js eslint.config.js
import { configs } from "@pelatform/eslint-config";

export default [
  ...configs.fast,
  {
    languageOptions: {
      parserOptions: {
        tsconfigRootDir: import.meta.dirname,
      },
    },
  },
];
```

### Monorepo

```js eslint.config.js
import { configs } from "@pelatform/eslint-config";

export default [
  ...configs.fast,
  {
    languageOptions: {
      parserOptions: {
        tsconfigRootDir: import.meta.dirname,
        project: ["./tsconfig.json", "./packages/*/tsconfig.json"],
      },
    },
  },
];
```

### Next.js

```js eslint.config.js
import { configs } from "@pelatform/eslint-config";
import globals from "globals";

export default [
  ...configs.fast,
  {
    languageOptions: {
      parserOptions: {
        tsconfigRootDir: import.meta.dirname,
        project: "./tsconfig.json",
      },
      globals: {
        ...globals.browser,
        ...globals.node,
      },
    },
  },
];
```

## Troubleshooting

<Accordion title="TypeScript Project Not Found">
  Ensure `tsconfig.json` is in the same directory as `eslint.config.js`:

  ```js eslint.config.js
  {
    languageOptions: {
      parserOptions: {
        tsconfigRootDir: import.meta.dirname,
      },
    },
  }
  ```
</Accordion>

<Accordion title="Type Checking Errors">
  Disable type-aware linting for non-strict projects:

  ```js eslint.config.js
  import { configs } from "@pelatform/eslint-config";
  import tsParser from "@typescript-eslint/parser";

  export default [
    ...configs.fast,
    {
      files: ["**/*.{js,ts}"],
      languageOptions: {
        parser: tsParser,
        parserOptions: {
          project: false,
        },
      },
    },
  ];
  ```
</Accordion>

## Migration

From legacy `.eslintrc` configuration:

1. Upgrade to ESLint 9.x
2. Replace `.eslintrc` with `eslint.config.js`
3. Use the new flat config format
4. Remove deprecated settings and plugins
